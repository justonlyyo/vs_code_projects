# 模块接口

-----

* **host**: 47.98.113.173
* **port**: 9519

## 对象说明

### 数据模型

字段名|中文名|类型|说明
:--:|:--:|:--:|:--:
id|主键|整数|唯一, 必须
name|模块名|字符串|必须
name_en|英文模块名|字符串|
code|编码|字符串|唯一,必须
version|版本号|字符串|固定,必须
url|路径|字符串|唯一,必须
has_entry| 是否入口模块|字符串|是否有独立的登录入口?1有,0没有
desc|备注/说明|字符串| 对模块的备注和说明内容
desc_en|英文备注|字符串|英文备注内容
creator|创建者id|整数|创建者id,必须,无需输入
last_user|最后修改者id|整数|最后的修改者id,必须,无需输入
create_time|创建时间|字符串|无需输入
last_time|最后修改时间|字符串|无需输入
children_ids|后代模块id|数组| 直接后代模块的id
parent_id|父模块id|整数|可为null,为null表示为顶层模块
hotel_groups|集团id|数组|拥有此(顶层)模块的功能的集团id
pages|页面|数组|一个终端模块(导航的末端)对应的页面(可能多个)

### 模型说明

* **has_entry** 是否拥有自己的登录入口.此字段表明对应的模块是否可以出现在登录后的入口选择菜单中.1表示有,0表示没有.
* **parent_id** 指向本模块的父模块.如果此字段为null,表明此模块为顶层模块
* **hotel_groups** 系统拥有的顶层模块在相对时间区间内是确定的.此字段是表明了那些集团拥有本模块的访问功能.
* **pages** 权限要细化到页面.也就是说,模块模块的终端连接页面,所有的权限在前端只和页面建立对应关系.(比如某个页面需要几个权限,这个信息需要在create阶段传递给前端)

## 接口约定

### 添加模块信息

* **url**: /v1/common/module/add
* **headers**: authorization 验证
* **method**: post
* **request args**:  

参数名|中文名|是否必须|参数类型|说明
:--:|:--:|:--:|:--:|:--:
name|模块名|Yes|字符串|模块的名称
code|模块代码|Yes|字符串|模块的code, 唯一
version|版本|No|字符串|版本号,默认 v1.0
url|模块链接|Yes|字符串|模块的链接
has_entry| 是否入口模块|Yes|字符串|是否有独立的登录入口?1有,0没有
desc|备注/说明|No|字符串| 对模块的备注和说明内容
desc_en|英文备注|No|字符串|英文备注内容
parent_id|父模块id|No|整数|可为null,为null表示为顶层模块

* **reqsponse type**": json
* **response": {"message": "success", "new_authorization": 新authorization}
* **respnse example**:

```javascript
{"message": "success"}
```

### 批量获取模块信息

* **url**: /v1/common/module/info_list
* **headers**: authorization 验证
* **method**: get/post
* **request args**:  

参数名|中文名|是否必须|参数类型|说明
